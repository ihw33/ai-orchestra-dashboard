AI Orchestra Dashboard - Product Requirements Document (PRD)

PROJECT OVERVIEW
================
The AI Orchestra Dashboard is a comprehensive management system designed to coordinate multiple AI CLI tools as a unified team. The system enables real-time collaboration, task assignment, and progress tracking through GitHub Issues integration and automated communication protocols.

CORE OBJECTIVES
===============
1. Orchestrate 4+ AI team members (Claude, Gemini, Codex, Cursor) as a cohesive unit
2. Automate task distribution and progress tracking via GitHub Issues
3. Provide real-time monitoring dashboard for all AI activities
4. Enable seamless communication between AI agents using AppleScript automation
5. Implement intelligent workflow management for complex multi-stage projects

TARGET USERS
============
- Project Managers (PM Claude) coordinating AI teams
- AI Team Members receiving and executing tasks
- Human stakeholders monitoring progress
- Development teams integrating AI collaboration workflows

TECHNICAL ARCHITECTURE
======================

Frontend (Next.js + TypeScript + Tailwind)
- Real-time dashboard displaying team status
- Task assignment interface
- Progress visualization
- WebSocket integration for live updates
- Multi-language support (Korean/English)

Backend (FastAPI + Python)
- GitHub API integration for Issues/PRs management
- WebSocket server for real-time communication
- Task orchestration engine
- Progress tracking and analytics
- AppleScript automation endpoints

AI Team Communication Layer
- iTerm2 session management and automation
- AppleScript-based message delivery system
- Mode detection (Shell vs AI mode for Gemini)
- Cursor app integration via keyboard shortcuts
- Automated confirmation and status reporting

TEAM MEMBER SPECIFICATIONS
==========================

PM Claude (Terminal)
- Role: Project manager and orchestrator
- Responsibilities: Task creation, assignment, monitoring, GitHub operations
- Tools: GitHub CLI, terminal commands, orchestration scripts
- Communication: Direct terminal access

Claude (iTerm Session 1)
- Role: Code review and documentation specialist
- Mode: AI mode permanently
- Specialization: Code quality, documentation, best practices
- Communication: AppleScript → iTerm session messaging

Gemini (iTerm Session 2)
- Role: Data collection and analysis specialist
- Mode: Toggleable Shell ($) ↔ AI (>) modes
- Specialization: Research, data processing, analytical tasks
- Special handling: Mode detection and switching automation

Codex (iTerm Session 4)
- Role: Code generation and automation specialist
- Mode: AI mode permanently
- Specialization: Code implementation, automation scripts
- Communication: AppleScript → iTerm session messaging

Cursor
- Role: VS Code-based AI editor
- Access: Cmd+K keyboard shortcut for AI chat
- Specialization: Interactive development, UI implementation
- Integration: AppleScript automation for message delivery

WORKFLOW SPECIFICATIONS
=======================

1. TASK ASSIGNMENT PROCESS
   a) PM creates GitHub Issue with detailed requirements
   b) System analyzes task and determines optimal AI assignee
   c) AppleScript delivers task notification to assigned AI
   d) AI confirms receipt and provides initial assessment
   e) Task begins with automatic progress tracking

2. PROGRESS MONITORING
   a) 5-minute intervals: File system monitoring for changes
   b) 30-minute intervals: Required progress reports from AI agents
   c) Real-time: GitHub Issue comment monitoring
   d) Continuous: Session health and mode detection

3. COMMUNICATION PROTOCOLS
   a) Standardized message formats for different AI types
   b) Confirmation requirements and timeout handling
   c) Escalation procedures for non-responsive agents
   d) Error handling and recovery mechanisms

4. QUALITY ASSURANCE
   a) Peer review assignments for critical tasks
   b) Automated testing integration
   c) Documentation requirements
   d) Final approval workflows

RISK MITIGATION
===============

Technical Risks:
- AppleScript automation failures → Fallback notification systems
- AI agent non-responsiveness → Timeout and reassignment protocols
- GitHub API rate limiting → Request queuing and caching
- Session management issues → Automatic recovery and restart

Operational Risks:
- Task complexity overestimation → Dynamic scope adjustment
- Communication bottlenecks → Parallel communication channels
- Quality degradation → Enhanced review processes
- Scalability limitations → Microservices architecture
