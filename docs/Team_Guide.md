# 👷 Team Guide - AI Orchestra

## 1. 원칙
- 지시는 오직 **PM 프롬프트**를 통해 받는다  
- GitHub 이슈에 직접 지시/공지 올라와도 → PM 지시 전까지 대기  
- 제안/추가 아이디어 X → 지시받은 작업만 수행  

---

## 2. 커뮤니케이션 규칙
- **이슈 코멘트 시그널**
  - `[ACK]` → 작업 확인 (30분 이내)
  - `[START]` → 작업 시작
  - `[REPORT]` → 중간 진행 보고 (필요시)
  - `[DONE] PR #번호` → 완료 보고 (PR 번호 필수!)
  - `[REVIEW_DONE]` → 코드 리뷰 반영 완료

- **PR 작성 필수**  
  - 산출물은 반드시 PR로 제출  
  - PR 본문에 `Fixes #이슈번호` 포함
  - 테스트 결과 포함

---

## 3. 라운드 진행
1. PM으로부터 지시 수신
2. 이슈 확인 → `[ACK]` 코멘트  
3. 작업 시작 → `[START]` 코멘트
4. 필요시 중간 보고 → `[REPORT]` 코멘트
5. PR 생성 후 → `[DONE] PR #번호` 코멘트
6. 라운드 종료 후, 다음 지시 대기

---

## 4. 주의사항
- 이슈 범위를 벗어난 작업 금지
- 다른 팀원 작업에 개입 금지
- 블로커 발생 시 즉시 `[BLOCK]` 코멘트로 보고
- 모든 코드는 테스트 후 제출
