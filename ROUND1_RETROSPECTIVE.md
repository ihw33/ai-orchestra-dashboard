## 😓 문제점 및 교훈

### 1. PM 준비 부족
**문제점:**
- Issue 내용이 너무 간략하여 팀원들이 작업 시작 후 수정
- Git 브랜치 전략 미수립
- 작업 순서와 의존성 고려 부족

**교훈:**
- Issue는 작업 시작 전에 완전히 준비
- 명확한 Git 워크플로우 정의 필요
- 작업 의존성 매트릭스 작성 필요

### 2. 통신 문제
**문제점:**
- Gemini Shell/AI 모드 전환 이슈 (해결됨)
- Cursor Cmd+K 단축키 작동 안 함
- AppleScript 한글 인코딩 문제

**교훈:**
- 각 AI 도구의 특성 사전 파악 필요
- 백업 통신 방법 준비
- 영어 기반 통신 우선

### 3. 보고 체계
**문제점:**
- 진행 중 보고(30분마다) 실제로 안 됨
- Issue 댓글 형식 불일치
- 완료 기준 불명확

**교훈:**
- 현실적인 보고 주기 설정
- 자동화된 모니터링 도구 필요
- 명확한 완료 정의(DoD)

### 4. 문서화
**문제점:**
- 너무 많은 문서 파일 생성 (혼란 야기)
- 중복되는 내용 많음
- 실제 사용되지 않는 문서들

**교훈:**
- 핵심 문서만 유지
- 단일 진실 공급원(SSOT) 원칙
- 문서도 코드처럼 관리
